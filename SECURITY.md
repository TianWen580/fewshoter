# Security Policy

## Reporting a Vulnerability

If you find a security issue, please use GitHub private vulnerability reporting for this repository.

Please include:

- A clear description of the issue
- Steps to reproduce
- Impact assessment
- Suggested mitigation (if available)

## Scope

Security reports for credential leaks, unsafe file handling, dependency vulnerabilities,
or remote service exposure are in scope.

## Response

Maintainers will acknowledge reports and prioritize fixes based on severity.
